This guide walks through a complete end-to-end workflow: from launching the CLI to having a working API client saved to disk.

<Steps>
  <Step title="Launch the CLI">
    Start the interactive REPL:

    ```bash
    reverse-api-engineer
    ```

    The banner displays your active SDK and model. The prompt shows the current mode in brackets — it starts in `agent` mode by default.

    ```
    [agent] > 
    ```

    Use **Shift+Tab** to cycle between modes: `agent` → `manual` → `engineer` → `collector`.

    | Mode | What it does |
    |---|---|
    | `agent` | Autonomous AI browser agent navigates and captures traffic automatically |
    | `manual` | You browse manually; AI generates the client when you close the browser |
    | `engineer` | Re-runs AI generation on a previous capture (no new browsing) |
    | `collector` | AI collects structured web data and exports to JSON/CSV |
  </Step>

  <Step title="Enter a task description">
    Type a plain-English description of what you want to capture. For this example, use the Apple Jobs careers page:

    ```
    [agent] > fetch all apple jobs from their careers page
    ```

    In **agent mode**, the CLI confirms that the agent will navigate autonomously:

    ```
    autonomous agent will navigate on its own
    ```

    In **manual mode**, you are prompted for an optional starting URL:

    ```
    [manual] > fetch all apple jobs from their careers page
     > url (Enter for none): https://jobs.apple.com
    ```

    <Tip>
      In agent mode, press **Ctrl+R** to fill the prompt with a random task suggestion. Press multiple times to cycle through ideas.
    </Tip>
  </Step>

  <Step title="Browse with the automated browser">
    A Chromium window opens. What happens next depends on your mode:

    **Agent mode** — the AI agent navigates the site autonomously based on your task description. It uses browser MCP tools to click, scroll, and interact without any input from you.

    **Manual mode** — you control the browser directly. Navigate to the pages you want to capture, interact with the UI to trigger the API calls you need, then close the browser when you are done.

    The browser captures all network traffic as a HAR (HTTP Archive) file in the background. You do not need to do anything special — just browse normally.

    <Note>
      Some websites use bot-detection techniques that may interfere with automated browsing. If you encounter issues, switch to `manual` mode (Shift+Tab) and browse the site yourself.
    </Note>
  </Step>

  <Step title="AI generates the API client">
    After the browser session ends, the AI analyzes the captured HAR file and generates production-ready code. You will see progress output in the terminal as files are written in real time.

    The generated code includes:
    - Authentication handling and CSRF/session token management
    - Clean function interfaces with type hints
    - Docstrings explaining each method and its parameters
    - Error handling with informative log messages
    - Pagination support where applicable

    Here is an excerpt from a real `api_client.py` generated from the Apple Jobs example:

    ```python api_client.py
    import requests
    from typing import Dict, List, Optional, Any
    from dataclasses import dataclass
    import logging

    logging.basicConfig(level=logging.INFO)
    logger = logging.getLogger(__name__)


    class AppleJobsAPI:
        """
        Client for interacting with the Apple Jobs API.

        Handles CSRF token management, session management with proper
        cookies, job search with filters and pagination, and error handling.

        Example usage:
            >>> client = AppleJobsAPI()
            >>> jobs = client.search_jobs(query="engineer", page=1)
            >>> for job in jobs:
            ...     print(f"{job.postingTitle} - {job.url}")
        """

        BASE_URL = "https://jobs.apple.com"
        API_BASE = f"{BASE_URL}/api/v1"

        def __init__(self, locale: str = "en-us"):
            self.locale = locale
            self.session = requests.Session()
            self._csrf_token: Optional[str] = None

            self.session.headers.update({
                "User-Agent": "Mozilla/5.0 ...",
                "Content-Type": "application/json",
                "browserlocale": locale,
            })

        def search_jobs(
            self,
            query: str = "",
            filters: Optional[Dict[str, Any]] = None,
            page: int = 1,
        ) -> List[dict]:
            """
            Search for job postings.

            Args:
                query: Search string (searches title and description)
                filters: Dict of filters, e.g. {"postLocation": ["postLocation-FRA"]}
                page: Page number (1-indexed)

            Returns:
                List of job dicts from the API response
            """
            self._ensure_csrf_token()
            payload = {
                "query": query,
                "filters": filters or {},
                "page": page,
                "locale": self.locale,
            }
            response = self.session.post(f"{self.API_BASE}/search", json=payload)
            response.raise_for_status()
            return response.json().get("res", {}).get("searchResults", [])
    ```
  </Step>

  <Step title="Find your output files">
    Generated scripts are saved in two locations:

    - `~/.reverse-api/runs/scripts/{run_id}/` — permanent storage, always kept
    - `./scripts/{task_name}/` — a local copy in your working directory with a readable name

    The local copy uses the descriptive folder name derived from your task description:

    ```
    ./scripts/apple_jobs_api/
    ├── api_client.py      # The API client with all methods
    ├── example_usage.py   # Runnable usage examples
    └── README.md          # Documentation and API summary
    ```

    To run the generated client immediately:

    ```bash
    cd ./scripts/apple_jobs_api/
    python example_usage.py
    ```

    <Tip>
      If the generated code needs refinement, switch to `engineer` mode (Shift+Tab), enter the `run_id` shown in the terminal output, and provide additional instructions. Example: `@id abc123 add pagination support`.
    </Tip>
  </Step>
</Steps>

## CLI commands

While inside the REPL, you can use these slash commands at any time:

| Command | Description |
|---|---|
| `/settings` | Configure model, SDK, agent provider, output language, and sync settings |
| `/history` | View past runs with timestamps, costs, and prompts |
| `/messages <run_id>` | View detailed LLM message logs from a specific run |
| `/help` | Show all available commands |
| `/exit` | Quit the CLI |
